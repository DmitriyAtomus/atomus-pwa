# Атом Электрика — ТЗ: цикл проектирования электрощита

Документ фиксирует рабочую логику модуля «Атом Электрика» (мастер + редактор AtomCAD).
Источник — рабочий процесс проектировщика. Версионируется вместе с кодом.

## 0. Принцип
Проект = **граф связей** аппаратов. У каждого элемента есть две стороны:
- **чем питается / откуда приходит сигнал** (вход) — источник: ввод, автомат, БП, выход контроллера;
- **что питает / что коммутирует** (выход) — потребители/нагрузки.

Схема (силовая, управления, аппараты) и спецификация **выводятся из этого графа**, а не рисуются вручную. Редактор понимает граф так же, как генератор → рисует цепочку.

### Граница шкафа
**Весь граф — это обвязка ВНУТРИ шкафа.** Клемма (Xn) — это **граница щита**:
- внутри: ввод → автоматы → коммутаторы (KM/KL/ТР) / БП / контроллер → **клеммы**;
- снаружи (за клеммой): внешнее оборудование (наружный блок, вентилятор, привод заслонки, датчик в поле). Оно **не рисуется аппаратом внутри**, а обозначается **подписью у клеммы** «→ Наименование».
Генератор строит **внутреннюю схему шкафа**; всё, что за клеммой — только адресная подпись.

```
ВВОД 3N~400В
   │
   ├─ QF (автомат) ── [коммутатор] ── Клемма ── Нагрузка
   │                    KM / KL / ТР / напрямую
   │
   └─ QF(упр.) ── БП(G) ─ +24В ─┬─ Контроллер ── (DO) ── катушка KM/KL
                                ├─ Сенсорная панель (RS-485)
                                └─ Датчики (AI)
```

## 1. Раздел / назначение щита
- Задаём обозначение и наименование: `ЩУ-001.010` и т.п. (раздел + номер → код документа `АГ.ЩУ-001/26`).
- Код идёт в основную надпись (штамп ГОСТ 2.104, форма 1) на всех листах.

## 2. Потребители (оборудование)
Для каждого: наименование, **напряжение** (220 / 380 / 24 / 36 В…), мощность/ток, кол-во.
- Указываем не только **питание**, но и **управление**: пример «Узел регулирования: привод, питание 24В, **управление 0–10В**».
- Тип задаёт способ коммутации (`needsContactor`/прямое/мотор/ТЭН/мотор-привод).
- НН-потребители (24/36В) питаются **от БП**, а не от ввода.

## 3. Датчики и сигналы
Тип сигнала: **NTC / Pt1000 / 4–20 мА / 0–10 В / дискретный**, кол-во.
- Ложатся на входы контроллера (AI/DI) и в сигнальный клеммник (по 2 клеммы на датчик: сигнал + общий).

## 4. Клеммы
Внешний интерфейс щита: список клемм с описанием, **редактируемый**.
- Авто-набор по составу: ввод X1, линии X2, датчики X5… Можно добавлять/править/менять порядок.
- В компоновке клеммы рисуются реальными размерами (IEK КПИ 2в по сечению), с зазором под кабель-канал.

## 5. Автоматы — распределение
- Собираются из состава: ток на каждом считается, номинал проверяется.
- Можно менять номинал, **объединять несколько линий на один автомат** или выносить обратно.
- Видно перегруз. Указываем «какой автомат кому подключается».

## 6. Вспомогательное (в щите)
Переключатели, магнитные пускатели (KM), твердотельные реле (ТР), промежуточные реле (KL),
**блоки питания (G)**, частотники, кнопки, лампы и т.д.
- У каждого — «**что коммутирует / что питает**» (targets): связь с нагрузкой(ами).
- Один источник (БП, автомат) может питать **несколько** приёмников (контроллер + панель + датчики).

## 7. Цепочка (граф) — главное
Указываем контроллер → строится цепочка управления и силовая:
- **Контроллер (DO)** → катушка **KM** → силовой контакт KM → … → **клемма** → нагрузка.
- Если нагрузку коммутирует **KL** (реле) — в силовой цепи стоит **контакт реле**, не контактор.
- Если **ТР** (0–10В) — после коммутатора стоит ТР (фаза + управление 0–10В от AO).
- **БП**: питается 220В (через выбранный автомат), питает выбранные приёмники (контроллер, панель, датчики 24В). Рисуется на силовом листе: QF → БП(L/N) → +24В/0В → клеммы → контроллер/датчики.
- Каждый элемент знает: **чем питается** (вход) и **что питает/коммутирует** (выход). Из этого редактор и генератор рисуют связи.

## 8. Обязательные правила оформления
- **Сечение кабеля** показывается **всегда** на каждом отводе (по номиналу автомата; ряд без 1,0 — минимум 1,5 мм²; слаботочка 0,75).
- **Фазы L1/L2/L3** — разными цветами (провода и подписи), легенда.
- **Клеммы** — IEK КПИ 2в по сечению; подключение сверху, обозначение снизу; зазор ≥5 см между рейками под кабель-канал; авто-подбор корпуса с учётом числа клемм; предупреждение «нужен корпус выше».
- **Штамп** — ГОСТ форма 1 (185×55), графы по ячейкам, обозначение по центру верхней строки.
- **Библиотека УГО**: загрузка по марке, по **PDF/фото** (ИИ распознаёт), **авто-отрисовка УГО ИИ** (`/api/atomcad/draw-ugo`), ручной **редактор УГО** (линия/пунктир/вывод/текст, фото-подложка). УГО хранится у аппарата и **подставляется на схему**.

## Листы на выходе
- **Лист 1 · Силовые цепи** — ввод, автоматы, коммутаторы (KM/KL/ТР), БП, клеммы нагрузок.
- **Лист 2 · Цепи управления** — контроллер, его I/O, катушки, питание 24В.
- **Лист 3 · Аппараты управления** — раскладка УГО аппаратов (кнопки/реле/ТР/БП…).
- **Спецификация** — перечень элементов (с клеммами КПИ по типоразмерам).
- **Компоновка** — расстановка в корпусе (DIN-рейки, клеммники, кабель-каналы).
